# AI混剪系统整体链路总览

## 一、全链路流程图

```text
┌─────────────────────────────────────────────────────────────────┐
│  STEP 1  爆款拆解                                                │
│                                                                  │
│  输入：视频字幕/转写文本                                          │
│     ↓                                                            │
│  【并行】                                                         │
│  ├── Prompt 1A：分析摘要  → viral_analysis 表                    │
│  └── Prompt 1B：分镜拆解  → viral_scenes 表                      │
│                                                                  │
│  模式：单一 Prompt × 2 并行                                       │
└─────────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────────┐
│  STEP 2  脚本写作                                                │
│                                                                  │
│  输入：                                                           │
│  - viral_analysis + viral_scenes（上一步）                        │
│  - 品牌知识库 + IP人物库（System Prompt 固定注入）                 │
│  - 用户勾选商品信息（动态拼入）                                    │
│  - 用户选择出镜IP + 目标平台                                       │
│     ↓                                                            │
│  【并行】                                                         │
│  ├── Prompt 2A：脚本摘要  → scripts 表                           │
│  └── Prompt 2B：分镜脚本  → script_scenes 表                     │
│       └── 每个分镜输出 material_tags + use_original_audio         │
│                                                                  │
│  模式：单一 Prompt × 2 并行                                       │
└─────────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────────┐
│  STEP 3  素材匹配                                                │
│                                                                  │
│  输入：script_scenes.material_tags                               │
│     ↓                                                            │
│  按 material_type 分两组处理：                                    │
│                                                                  │
│  【第一组 并行】product_display / product_explain                 │
│    关键词精准匹配，命中→选用，未命中→mark_unmatched               │
│     ↓（完成后）                                                   │
│  【第二组 并行】effect_demo / ip_lifestyle                        │
│    向量语义搜索，返回Top5备选                                      │
│     ↓                                                            │
│  冲突检测（同素材被多分镜选中→次优备选升主）                        │
│     ↓                                                            │
│  → material_matches 表 + match_summary 表                        │
│                                                                  │
│  模式：Agent（分组并行 + 工具调用）                                │
│  工具：search_materials / get_material_detail / mark_unmatched   │
└─────────────────────────────────────────────────────────────────┘
                          ↓
              用户确认素材（可手动替换备选）
                          ↓
┌─────────────────────────────────────────────────────────────────┐
│  STEP 4  混剪                                                    │
│                                                                  │
│  输入：script_scenes + material_matches + BGM库 + 爆款风格信息    │
│     ↓                                                            │
│  【并行】                                                         │
│  ├── Prompt 4A：基础剪辑配置                                      │
│  │     转场/特效智能匹配 + BGM同款/相似匹配                        │
│  │     use_original_audio=true → 素材原声，不生成配音             │
│  │     → 写入基础配置（共用）                                      │
│  │                                                               │
│  └── 后端素材组合算法：生成 N 套素材组合                           │
│        版本1=全主素材，版本2起轮换备选，差异最大化                  │
│     ↓（等两者完成）                                               │
│  Prompt 4B：口播文案差异化改写（一次输出 N 个版本）                │
│     ↓                                                            │
│  后端合并引擎（并行生成 N 个最终配置）：                           │
│    版本i = 基础配置 + 素材组合i + 文案版本i + 随机微调参数         │
│     ↓                                                            │
│  → video_configs 表（N条）+ video_scenes 表 + video_bgm_tracks 表│
│     ↓                                                            │
│  并行渲染 N 条视频                                                │
│                                                                  │
│  模式：单一 Prompt × 2 串行 + 后端算法                            │
└─────────────────────────────────────────────────────────────────┘
```

## 二、各步骤 AI 模式汇总


| 步骤      | AI 模式        | 调用次数      | 并行策略       |
| ------- | ------------ | --------- | ---------- |
| 爆款拆解    | 单一 Prompt    | 2次        | 并行         |
| 脚本写作    | 单一 Prompt    | 2次        | 并行         |
| 素材匹配    | Agent + 工具调用 | 动态（按分镜数）  | 分组并行       |
| 混剪-基础配置 | 单一 Prompt    | 1次        | 与素材组合算法并行  |
| 混剪-文案改写 | 单一 Prompt    | 1次（输出N版本） | 串行（依赖基础配置） |


---

## 三、异常处理与阻断逻辑


| 异常情况           | 处理方式              | 是否阻断后续步骤  |
| -------------- | ----------------- | --------- |
| 爆款拆解 JSON 解析失败 | 重试一次，仍失败提示用户重新上传  | 是         |
| 脚本写作输出缺少必要字段   | 后端补默认值，提示用户人工检查   | 否         |
| 素材匹配 unmatched | 标记补拍提醒，允许跳过继续混剪   | 否（可跳过）    |
| 混剪基础配置生成失败     | 重试一次，仍失败回退到手动配置   | 是         |
| 口播文案改写版本数不足    | 已生成的版本正常渲染，不足部分提示 | 否         |
| 视频渲染失败         | 保留配置，支持单条重新渲染     | 否（其他版本继续） |

## 六、各步骤耗时估算

```text
STEP 1 爆款拆解      ~5-10s     （两个 Prompt 并行，取慢者）
STEP 2 脚本写作      ~8-15s     （两个 Prompt 并行，取慢者）
STEP 3 素材匹配      ~3-8s      （分组并行 + 工具调用）
STEP 4 混剪配置      ~10-15s    （Prompt 4A∥素材算法 + Prompt 4B 串行）
       渲染          ~N×2min    （N条并行，取最慢者，约2-3min）
───────────────────────────────
总计（不含渲染）      ~26-48s
总计（含渲染）        ~3-5min
```
